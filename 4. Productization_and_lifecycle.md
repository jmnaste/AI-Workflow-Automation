## 9. Glossary of Technical Terms and Acronyms
| Term | Definition |
|------|-------------|
| **AI Workflow** | Sequence of automated tasks enhanced with AI reasoning. |
| **LangGraph** | Framework for building AI agent graphs and stateful reasoning workflows. |
| **FastAPI** | Python framework for serving APIs and workflow endpoints. |
| **n8n** | Open-source automation tool for connecting apps and APIs. |
| **RAG (Retrieval-Augmented Generation)** | AI technique combining document retrieval and text generation. |
| **PGVector** | PostgreSQL extension for storing and querying vector embeddings. |
| **POC (Proof of Concept)** | Early demonstration of a product or feature to validate feasibility. |
| **SLA (Service Level Agreement)** | Contract defining service reliability expectations. |
| **SLO (Service Level Objective)** | Quantitative performance targets (e.g., uptime, latency). |
| **HITL (Human-in-the-Loop)** | Step where humans review or correct AI outputs. |
| **ROI (Return on Investment)** | Financial metric comparing cost vs benefit of automation. |
| **p95 Latency** | Response time under which 95% of requests complete. |
| **CI/CD** | Continuous Integration / Continuous Deployment pipeline for automation. |
| **Observability** | Monitoring system behavior through metrics, logs, and traces. |
| **Template Extraction** | Process of generalizing a project artifact into a reusable component. |
| **Productization** | Turning custom-built solutions into standardized, repeatable products. |
| **Monorepo** | Single repository containing multiple project components. |
| **Semantic Commit** | Git convention encoding the purpose of a commit (e.g., feat, fix, docs). |
| **Golden Set** | Canonical set of evaluation cases for validating workflow reliability. |
